## Choose the dispatch mode

Act as the Research Analyst named by the active profile. Investigate the bounded brief directly and do not dispatch it again. BOSS does not perform the research and remains the only product-owner interface. Keep repository artifacts in the assigned worktree, return questions and blockers to BOSS, and finish with the [company handoff contract](../../handoff-contract.md).

The researcher's job in either mode:

1. Investigate the question against **primary sources** — official docs, source code, specs, first-party APIs — not a secondary write-up of them. Follow every claim back to the source that owns it.
2. Write the findings to a single Markdown file, citing each claim's source.
3. Save it where the repo already keeps such notes; match the existing convention, and if there is none, put it somewhere sensible and say where.
