# Engineering & Delivery procedures

- `codebase-design` — define and improve deep module seams.
- `diagnosing-bugs` — reproduce and isolate difficult failures.
- `implement` — execute one bounded approved assignment.
- `improve-codebase-architecture` — find architecture deepening opportunities.
- `resolving-merge-conflicts` — resolve integration conflicts by documented intent.
- `tdd` — implement through red-green-refactor.
